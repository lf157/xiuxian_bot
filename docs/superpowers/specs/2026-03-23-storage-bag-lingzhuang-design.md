# 储物袋与灵装入口拆分设计

**日期:** 2026-03-23  
**范围:** `adapters/telegram/bot.py` Telegram 面板交互

## 背景
当前实现中，装备已在代码层作为 `equipbag` 单独分页，但玩家仍需先进入“背包”才能触达装备入口。你已确认：
- 主界面采用“双入口”：`🎒 储物袋` + `👕 灵装`
- 储物袋只显示可堆叠物品，不再出现装备信息
- 装备名称统一为“灵装”

## 目标
1. 将“装备背包”从“背包内入口”升级为“主界面一级入口”。
2. 将“背包”更名为“储物袋”，并只保留丹药/材料等可堆叠物品操作。
3. 将装备相关文案与按钮统一改为“灵装”。
4. 保持现有装备能力不变：装备、强化、分解、已装备查看/卸下。

## 方案
采用最小改造策略（你已确认的方案1）：
- 保留现有回调链路 `equipbag_*` 作为底层能力，新增主入口 `equipment` 作为别名入口。
- 主菜单按钮改为：`🎒 储物袋`、`👕 灵装`。
- `bag` 页面文字与按钮去装备化：
  - 不显示“装备数量”
  - 不显示“装备背包”按钮
  - 空储物袋提示不再引导到装备页
- 装备页面统一改名“灵装”，并从页面底部返回“储物袋”。

## 数据与接口影响
- 不改数据库结构。
- 不改后端 API（仍调用 `/api/items`, `/api/equip`, `/api/enhance`, `/api/decompose`, `/api/unequip`）。
- 仅改 Telegram UI 层文案、按钮和回调分发。

## 兼容性
- 旧回调 `equipbag_*` 保留，避免历史消息按钮失效。
- 新入口 `equipment` 直接复用装备分页逻辑。

## 验收标准
1. 主菜单出现“储物袋/灵装”双入口。
2. 储物袋页没有“装备数量”“装备背包”入口。
3. 灵装页标题与跳转文案统一为“灵装”。
4. 装备/强化/分解/已装备流程可达。
